A container packages an application with the environment it needs. This reduces the familiar “works on my machine” problem.

## Write a Dockerfile

A Dockerfile describes a reproducible image. Keep it small, copy only needed files, and use a `.dockerignore` file.

## Run one service first

Build and run a simple application before adding databases or multi-container orchestration.
