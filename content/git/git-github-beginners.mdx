Git records the history of a project; GitHub hosts that history and makes collaboration easier.

## Work in small commits

Make each commit represent one understandable change. A clear message makes the project history useful to future you.

## Use branches for changes

Create a branch before a feature or fix. Open a pull request to review the work before merging it.
